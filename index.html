TCP Sidechannel Fingerprinting

by Patrick Beart

This page makes HTTP requests to a number of local addresses and local network addresses at various ports, exploiting the fact that although the Access-Control-Allow-Origin header can restrict code from another origin from receiving an HTTP response, whether the requests complete can still be determined from how long it takes.

Since a request completing before timing out only requires that the remote port sent something before the timeout (or more precisely, sent something invalid or a whole valid response), as long as the remote TCP port responds to invalid data (in this case an HTTP request) with some data (again, more precisely, data that doesn't happen to be some of but not all of a valid HTTP response), listening TCP ports can be detected by measuring how long the request takes to complete.
